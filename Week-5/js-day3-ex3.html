In this exercise I used the fetch method to make a GET, PUT, and POST request to my endpoint for a ressource. The fetch method is pr default a GET request, so when making a POST or PUT request I must supply an options parameter besides the URL. The options parameter specifies the metod, the headers and the body. the options parameter is passed as a JSON object. if the body doesnt align with the expected request and the endpoint, an error will occur (status 400 Bad request) the headers-object specifies content type in this case but could include other headers with additional information. the URL specifies the ressource endpoint for the request. Remember to augment the base-url to include the Id of the single ressource when making a POST, PUT or PATCH request.
